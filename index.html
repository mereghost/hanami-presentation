O que é Hanami?

Hanami

é um framework web completo para Ruby

também a prática de apreciar a beleza do florescer das cerejeiras

O que ele oferece?

  1. Velocidade
  2. Segurança
  3. Funcionalidade e leveza
  4. Simplicidade
  1. O stack é menor e mais rápido. CDNs from the gound up.
  2. Content Security Policy (XSS), X-Frame headers (quem pode abrir em frames)
  3. 100nas funcionalidades, 60% menos memória.
  4. Diferenciar simples de familiar
Seu framework não é sua aplicação.
The web is just a delivery mechanism!

Controllers & Actions

Controllers

São apenas um modulo, um namespace.


module Web::Controllers::AwesomeClassName
  # here be dragons
end
Errr... e como ficam as actions?

Actions

São cidadãos de primeira classe.


module Web::Controllers::AwesomeClassName
  class Index
    include Web::Action

    def call(params)
      # here be dragons
    end
  end
end

Mas qual a vantagem disso?

Falar do call como ponto de referencia

Testabilidade


RSpec.describe Web::Controllers::AwesomeClassName::Index
  let(:action) { Web::Controllers::AwesomeClassName::Index.new }
  subject(:response) { action.call({}) }

  it 'does awesome things'
end

Só mostra o que os outros precisam ver


module Web::Controllers::AwesomeClassName
  class Index
    include Web::Action
    expose :inst_var #algo que a view precise ver
  end
end
Mas eu chamo um método sempre a cada request!

module Web
  module MyReusableBehaviour
  def self.included(action)
    action.class_eval do
      before :behaviour
      expose :some_object
    end
  end
  private def behaviour; @some_object = "Nothing to see here"; end
end

module Web::Controllers::AwesomeClassName
  class Index
    include Web::Action
    include Web::MyReusableBehaviour
  end
end

Views

  end
end

Novamente, cidadãos de primeira classe

São como presenters

E pelo mesmo motivo!


RSpec.describe Web::Views::AwesomeClassName::Index
  let(:exposures) { Hash[stuff: %w(cool awesome nifty)] }
  let(:template) do
    Hanami::View::Template.new('path/to/template.html.erb')
  end

  let(:view) do
    Web::Views::AwesomeClassName::Index.new(template, exposures)
  end

  let(:rendered) { view.render }

  it 'autoescape all strings' do
    expect(view.title).to eq('Hanami &amp; Ruby')
  end
end

Mappings

É como ligamos os repositorios, ao DB e às entities


Hanami::Model.configure do
  adapter type: :sql, uri: ENV['DATABASE_URL']
  migrations 'db/migrations'
  schema 'db/schema.sql'
  mapping do
    collection :awesome_things do
      entity AwesomeThing
      # repository AwesomeThingRepository
      attribute :id, Integer
      attribute :coolness_index, Integer
      attribute :created_at, DateTime
      attribute :updated_at, DateTime
    end
  end
end

E não é só isso:
